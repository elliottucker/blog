      <title>Downloading a YouTube playlist to MP3 files</title>
      <link>https://elliottucker.net/2014/downloading-a-youtube-playlist-to-mp3-files</link>
      <guid>https://elliottucker.net/2014/downloading-a-youtube-playlist-to-mp3-files</guid>
      <pubDate>Thu, 28 Aug 2014 21:07:46 +0000</pubDate>
      <description><![CDATA[<div>
        <p class="reading">
        <span class="vague">3 min read </span>
    </p>
    
<p><a href="https://elliottucker.net/content/all/?q=%23Downloading" class="p-category">#Downloading</a>a YouTube playlist to MP3 files</p>

<p>Now the kids are back to school we're more strict on device time at night however my daughter likes to fall asleep listening to the weird video game music she's favourited on YouTube, so she asked if they could be downloaded to her old iPod.  Here's how I did it on OSX.</p>

<p>We're going to use Python, the Google YouTube API and the rather cool youtube-dl, that can download and convert YouTube vidoes to audio.</p>

<p>_Assumptions:  You have Python, Brew and PIP and are using OSX...like me.  If you're not using Brew and Pip on OSX, you should be._</p>

<p>## Install the prequisites</p>

<p>	pip install --upgrade google-api-python-client youtube-dl requests<br />	brew install ffmpeg </p>

<p>## Set yourself up with a Google API account</p>

<p>You need a Google account and an API key.  You need this to get the list of videos IDs in a playlist.  Follow these steps at <a href="https://developers.google.com/youtube/v3/getting-started#before-you-start">https:/<wbr></wbr>/<wbr></wbr>developers.google.com/<wbr></wbr>youtube/<wbr></wbr>v3/<wbr></wbr>getting-started#before-you-start</a>.</p>

<p>You'll need you API key later.</p>

<p>## Find the playlist ID</p>

<p>On YouTube, click on the name of the playlist, either on your own or someone elses YouTube page.  The URL will include the playlist id, e.g, ```<a href="https://www.youtube.com/playlist?list=abcd%60%60%60">https:/<wbr></wbr>/<wbr></wbr>www.youtube.com/<wbr></wbr>playlist?list=abcd```</a>.  You need to make a note of the ID after list=</p>

<p>## Write the script that downloads the files</p>

<p>Replace key and playlistid with the key and the id you've just got above.</p>

<p>	#!/usr/bin/env python </p>

<p>	import requests<br />	from subprocess import call</p>

<p>	key = "<key>"<br />	playlistId = "<playlistid>"<br />	url = "https://www.googleapis.com/youtube/v3/playlistItems"<br /><br />	vidIds = []<br /><br />	r = requests.get("{}?key={}&part=snippet&playlistId={}&maxResults=50".format(url, key, playlistId))<br /><br />	for item in r.json()['items']:<br />	  vidIds.append(item['snippet']['resourceId']['videoId'])<br /><br />	nextPageToken = r.json()["nextPageToken"]<br /><br />	while nextPageToken is not None:<br />	  r = requests.get("{}?key={}&part=snippet&playlistId={}&maxResults=50&pageToken={}".format(url, key, playlistId, nextPageToken))<br /><br />	  for item in r.json()['items']:<br />	    vidIds.append(item['snippet']['resourceId']['videoId'])<br /><br /><br />	  nextPageToken = r.json().get("nextPageToken", None)<br /><br />	print "vid ids: {}".format(len(vidIds))<br />	for vid in vidIds:<br />	  print "getting {}".format(vid)<br />	  call(["youtube-dl", "https://www.youtube.com/watch?v={}".format(vid), "-x", "--audio-format=mp3"])<br /><br /><br />Save and run this...it should download the vids and convert to MP3s in the directory you're running it in.<br /><br />## Add ID3 tags<br /><br />You're going to want to separate all these nasty untagged MP3s into it's own album, so tag them like so<br />	brew install id3lib<br />	id3tag --album="Playlist Name" --artist="Various"<br /></playlistid></key></p>

      <title>Setting up Known with Nginx #indieweb</title>
      <link>https://elliottucker.net/2014/setting-up-known-with-nginx-indieweb</link>
      <guid>http://elliottucker.net/2014/setting-up-known-with-nginx-indieweb</guid>
      <pubDate>Fri, 22 Aug 2014 20:02:58 +0000</pubDate>
      <description><![CDATA[<div>
        <p class="reading">
        <span class="vague">6 min read </span>
    </p>
    
<p>[Known](<a href="http://withknown.com">http:/<wbr></wbr>/<wbr></wbr>withknown.com</a>) is the soon to be released open source blogging platform that has built-in "IndieWeb" features such as webmentions, POSSE (Publish On your Own Site Syndicate Elsewhere) and the ability to use yor own site as login mechanism for other IndieWeb compliant sites. For more info on IndieWeb check out <a href="http://indiewebcamp.com">http:/<wbr></wbr>/<wbr></wbr>indiewebcamp.com</a>.  </p>

<p>Known is due to be released "this summer" but the [pre-releases source code is available on GitHub](<a href="https://github.com/idno/idno">https:/<wbr></wbr>/<wbr></wbr>github.com/<wbr></wbr>idno/<wbr></wbr>idno</a>) so I thought I'd have a tinker. Much of my tinkering is likely to be made redundant by the time they release but never mind, it's all good practice.  </p>

<p>_Assumption: You're a sysadmin or other kind of Linux expert - and we're in Ubuntu/Debian land here._</p>

<p>Known requires the use of a web server software that can do URL rewriting and Known recommend using Apache and mod_rewrite. Other requirements are MongoDB or MySQL and PHP 5.4 or above. I've been using Apache for so long that I'm frankly bored of it and I try and use Nginx where possible. The primary challenge then is converting the Apache URL rewriting rules provided by Known in the .htaccess document into Nginx compatible rules. In addition I found that Known needs good caching in place and I'll talk a bit about that too.</p>

<p>##Getting started.</p>

<p>I'd suggest making use of a [Vagrant](<a href="http://vagrantup.com">http:/<wbr></wbr>/<wbr></wbr>vagrantup.com</a>) box and [Python Fabric](<a href="http://www.fabfile.org/">http:/<wbr></wbr>/<wbr></wbr>www.fabfile.org/<wbr></wbr></a>) to automate some (all?) of the steps. This is good devops practice anyway and once you start doing things this way it's hard to stop. If you're looking for a virtual server at a good price give [Digital Ocean](<a href="http://digitalocean.com">http:/<wbr></wbr>/<wbr></wbr>digitalocean.com</a>) a look.</p>

<p>For my setup I'm using Ubuntu 14.04 LTS and I'll assume you are too. I'm using mongodb as recommended by Known as there are zero extra steps required in setting up your backend DB, and I'll be using Nginx with php-fpm.</p>

<p>First install the following packages:</p>

<p>* mongodb<br />* nginx<br />* php5<br />* php-pear<br />* php5-dev<br />* php5-gd<br />* php5-curl<br />* php5-fpm<br />* php5-xmlrpc<br />* php5-mongo<br /><br />Then check out the Known source from <a href="https://github.com/idno/idno">https:/<wbr></wbr>/<wbr></wbr>github.com/<wbr></wbr>idno/<wbr></wbr>idno</a> into /var/www.  Use recurssive checkout as there are submodules in the project:<br />```git clone --recursive <a href="https://github.com/idno/idno">https:/<wbr></wbr>/<wbr></wbr>github.com/<wbr></wbr>idno/<wbr></wbr>idno</a> /var/www```</p>

<p>##Slight bit of code change</p>

<p>At this point in time (2014-08-22) there is a bit of code change to do which I've submitted to Known.  By the time you read this you might not even need to bother. In the file Idno/Pages/File/View.php there is an Apache specic function call, ```$headers = apache_request_headers();```.  This won't work with Ngnix so you need to apply a diff.</p>

<p>This should do it...</p>

<p>	cd /var/www/idno <br />	curl -O <a href="https://gist.githubusercontent.com/elliottucker/ab5a8d9c12a062a839ff/raw/f86126a1d6945fa9cb5a173e8c44d4b3aa6a3b23/known_view.diff">https:/<wbr></wbr>/<wbr></wbr>gist.githubusercontent.com/<wbr></wbr>elliottucker/<wbr></wbr>ab5a8d9c12a062a839ff/<wbr></wbr>raw/<wbr></wbr>f86126a1d6945fa9cb5a173e8c44d4b3aa6a3b23/<wbr></wbr>known_view.diff</a> | git apply -</p>

<p>##Set up Known URL rewriting with Nginx</p>

<p>Here's part the .htaccess file provided in the Known source that we want to work on Nginx:</p>

<p><ifmodule mod_rewrite.c><br />	    RewriteEngine on<br />	    RewriteRule ^(\.well\-known.*)$ /index.php/$1 [L]<br />	    RewriteRule ^\..* - [F]<br />	    RewriteCond %{REQUEST_FILENAME} !-f<br />	    RewriteCond %{REQUEST_FILENAME} !-d<br />	    RewriteCond $1 !^(index\.php)<br />	    RewriteRule ^(.*)$ /index.php/$1 [L]<br /></ifmodule></p>

<p>With some mucking about I managed to translate this to Nginx rules. Include this in the Nginx "server" block config. You'll need to change server_name to the hostname or IP of your server. I've only one site on the server so I've got this in /etc/nginx/sites-available/default.</p>

<p>	server {<br />	  listen 80;</p>

<p>	  root /var/www;<br />	  index index.php;</p>

<p>	  server_name example.com;<br />	  client_max_body_size 20M;<br /><br />	  location / {<br />      try_files $uri $uri/ /index.php?$query_string;<br />	  }<br /><br />	  location ~ \.php$ {<br />      fastcgi_split_path_info ^(.+\.php)(/.+)$;<br />      fastcgi_pass unix:/var/run/php5-fpm.sock;<br />      fastcgi_index index.php;<br />      include fastcgi_params;                <br />      fastcgi_param PHP_VALUE "cgi.fix_pathinfo=0 \n upload_max_filesize=1000M \n post_max_size=1080M";<br />	  }</p>

<p>	  location ~ /\.ht {<br />      deny all;<br />	  }<br />	  location ~* \.(js|css)$ {<br />      expires max;<br />      log_not_found off;<br />	  }</p>

<p>	  location ~* \.(xml|ini)$ {<br />      deny all;<br />	  }</p>

<p>	}</p>

<p>I know it looks a lot bigger but note this is the config for all of the site, not just the URL rewriting part.  In fact, for all intents and purposes the Apache rewriting config can be translated to just:</p>

<p>	location / {<br />		try_files $uri $uri/ /index.php?$query_string;<br />	}</p>

<p>Yes, Nginx is cool, once you get your head around it.</p>

<p>At this point you should be good to start up Nginx and point your browser to your host or IP address...if not, sorry, this is when you need to be a sysadmin or ask on <a href="https://elliottucker.net/content/all/?q=%23indiewebcamp" class="p-category">#indiewebcamp</a> for help.</p>

<p>##Setting up caching with Varnish.</p>

<p>I noticed that none of the images stored on the DB were caching in my browser, and this was very noticible on the large background image I've set up in the theme. I might be wrong, and it could be a side effect of using Nginx, but it looks like Known is forcing no cache for all requests. Having never used Varnish before I decided that this was the man for the job, no other reason.</p>

<p>First of install the 'varnish' package from apt.  Currently Ubuntu has Varnish 3 rather than 4 but this will do us.</p>

<p>Varnish will be listening on port 80 so we need to change the Nginx port to something else, e.g, 8080 and to have it only listen on locally, not on the public internet.  Change the 'listen' config and then restart Nginx:</p>

<p>	listen 127.0.0.1:8080;</p>

<p>To get Varnish to listen on port 80, change /etc/default/varnish to be the following:<br /><br />	DAEMON_OPTS="-a :80 \<br />             -T localhost:6082 \<br />             -f /etc/varnish/default.vcl \<br />             -S /etc/varnish/secret \<br />             -s malloc,256m"<br />	START="Yes"</p>

<p>You now want to set up the rules in Varnish to know where to find the backend site you have running on Nginx port 8080.  Replace /etc/varnish/default.vcl with the following.</p>

<p>	backend default {<br />	    .host = "127.0.0.1";<br />	    .port = "80";<br />	}</p>

<p>And in the same file add the following rule.  Basically removing any HTTP response headers that stop caching and setting the age for all cached items to 1d for any request starting /file/.</p>

<p>	sub vcl_fetch {<br />	    if (req.url ~ "^/file/") {<br />	        remove beresp.http.cache-control;<br />	        set beresp.http.Cache-Control = "public, max-age=1d";<br />	        remove beresp.http.Pragma;<br />	        set beresp.ttl = 1d;<br />	    }<br />	}</p>

<p>Restart Varnish and you should be good to go to your site and start setting it up. It's worth playing with Varnish now to see if you improve caching and performance.  This is something I haven't done and there's a very good chance the above config is far from optimal.  Let me know your results in comments or at <a href="https://elliottucker.net/content/all/?q=%23indiewebcamp" class="p-category">#indiewebcamp</a>.</p>

<p>##Next Steps</p>

<p>Now you have your site up and running (you do don't you?) try adding some plugins.  Check these out from <a href="https://github.com/idno">https:/<wbr></wbr>/<wbr></wbr>github.com/<wbr></wbr>idno</a> and stick them in the IdnoPlugins directory. I think there's an issue with the Twitter plugin at the moment where I couldn't get it to save the authentication.  Clear your browser session if that happens.  It might help.</p>
</div>]]></description>

      <title>Using gspread to populate a Google Spreadsheet from a multi dimensional dictionary</title>
      <link>https://elliottucker.net/2014/using-gspread-to-populate-a-google-spreadsheet-from-a-multi</link>
      <guid>http://www.elliottucker.net/2014/using-gspread-to-populate-a-google-spreadsheet-from-a-multi</guid>
      <pubDate>Thu, 21 Aug 2014 06:37:54 +0000</pubDate>
      <description><![CDATA[<div>
        <p class="reading">
        <span class="vague">3 min read </span>
    </p>
    
<p>I'm often asked to knock up quick transaction reports and in the past I'd normally automate sending these by email or having them exported to an Excel sheet in a shared directory.  As a company that uses Google Apps it made sense to have these reports created as a Google Spreadsheet instead.  Thanks to the excellent [gspread](<a href="https://github.com/burnash/gspread">https:/<wbr></wbr>/<wbr></wbr>github.com/<wbr></wbr>burnash/<wbr></wbr>gspread</a>) Python library this is extremely easy.</p>

<p>We'll create a Spreadsheet with multiple worksheets, one for each year in the report.  Worksheets will be made up of rows of months a columns of clients, with cells being a random value.</p>

<p>First off we'll make a two dimensional dictionary data to play with.  You can safely ignore test_data() but it gives a useful example of the kind of data I might see.</p>

<p>		from collections import defaultdict         # only required for <br />		from random import random                   # test data</p>

<p>		import gspread</p>

<p>		def test_data():</p>

<p>			months = ['Jan','Feb','Mar','Apr,','May','Jun','Jul']<br />			clients = ['Client 1','Client 2','Client 3','Client 4']</p>

<p>			data = defaultdict(dict)<br />			for month in months:<br />				for client in clients:<br />								data[month][client] = (int(random()*100),int(random()*100))</p>

<p>			return data</p>

<p>These should leave us with a dictionary with months and clients for keys with a list of random numbers but no particular order.</p>

<p>	&gt;&gt;&gt; data['Jan']['Client 1']<br />	(42,34)</p>

<p>Create a class to do the grunt work of creating worksheets and updating cells.  The class will take your Google username and password, and the name of a previously created Google Spreadshet.</p>

<p>Define a method suitable for your data - obviously this will be determined by your needs but the below example would be a common simple report.  Here I'm going to have a method that updates a single year's worksheet with the data defined in the above test data.  It will loop through the months, then the clients and create a row with the data "month,client,value 1, value 2"</p>

<p>	class Sheet:<br />			def __init__(self,username,password,spreadsheet):<br />					gc = gspread.login(username,password)<br />					self.spreadsheet = gc.open(spreadsheet)<br />					self.rowcount = {}</p>

<p>			def update_year(self,year,data):</p>

<p>					rowcount = 2  # Start at 2 to leave room for headers</p>

<p>					# Check for an existing worksheet for the year and create one if not<br />					# Each sheet is a year<br /><br />					try:<br />							worksheet= self.spreadsheet.worksheet(year)<br />					except gspread.WorksheetNotFound:<br />							self.spreadsheet.add_worksheet(year,100,5)<br />							worksheet = self.spreadsheet.worksheet(year)</p>

<p>					# Loop through the months in the data dictionary and build<br />					# a row to add to the worksheet"""</p>

<p>					for month in data.keys():</p>

<p>							for client in data[month].keys():<br /><br />									# a row of data to add to the spreadsheet.<br />									row = (month,client,data[month][client][0],data[month][client][1])</p>

<p>									cell_list = worksheet.range("A%s:D%s" % (rowcount,rowcount))</p>

<p>									# cell_list now containts a list of cell objects. Use map and a tiny<br />									# function (setcell) to update the list of cell objects with a row of values.</p>

<p>									map(self.setcell,cell_list,row)<br />									worksheet.update_cells(cell_list)<br />									# move on a row in the sheet.<br />									rowcount += 1</p>

<p>			def setcell(self,c,v):<br />					c.value = v</p>

<p>					sheet.update_year(year,data)</p>
